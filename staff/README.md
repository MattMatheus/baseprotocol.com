# Staff Directory

The Base Protocol staff directory gives agent responsibilities memorable names. The names are not fictional employees. They are routing handles for work contracts.

Use them when the user wants to direct work naturally:

- "Ask Clara to update the docs."
- "Get Ada to check architecture."
- "Have Iris verify this before release."
- "Ask Rowan to review the migration plan."

## Roles

| Name | Role | Primary Output |
| --- | --- | --- |
| Maya | Product Manager | Refined backlog item, bounded acceptance criteria, queue placement, state transition recommendation. |
| Ada | Software Architect | Architecture decision, rationale, alternatives, risks, follow-on implementation paths. |
| Iris | QA Engineer | Explicit verdict, defect list, evidence summary, next-state recommendation. |
| Theo | SDET | Test strategy, coverage map, automation priorities, known gaps. |
| Clara | Technical Writer | Updated documentation, clarified terminology, identified documentation gaps. |
| Nia | SRE | Reliability assessment, operational risks, required controls, release recommendation. |
| Rowan | Database Expert | Schema or storage recommendation, migration guidance, data risk summary. |

## Tone

Keep the staff layer useful and restrained. Names should make delegation easier, but the work should remain inspectable, factual, and grounded in the repository.
